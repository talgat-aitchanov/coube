1. Добавить контроллеры с base path /api/admin, для управления тарифами
2. Добавить jwt авторизацию для админских контроллеров
3. Добавить кэширование для ранее вычисленных расчетов, но с учетом TTL (Time To Live) для обеспечения актуальности данных
4. Добавить кэширование тарифов, чтобы не дергать базу данных при каждом запросе, но обновлять его регулярно (например, раз в сутки) или по изменению тарифов
5. Для consistency необходимо добавить pessimistic lock при обновлении тарифов, чтобы избежать гонок при одновременном обновлении. Это может быть реализовано с помощью @Lock(LockModeType.PESSIMISTIC_WRITE) в JPA репозитории.
6. Добавить Redis/Zookeepr для распределенной блокировки при деплое множественных экземпляров приложения, чтобы избежать конфликтов при обновлении тарифов в распределенной среде. Это обеспечит, что только один экземпляр приложения будет обновлять тарифы в определенный момент времени.
7. После обновления тарифов необходимо в админской панели получить уведомление об успешном обновлении путем SSE
